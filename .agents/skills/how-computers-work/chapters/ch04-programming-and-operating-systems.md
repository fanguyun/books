# ch04 编程与操作系统

这一部分对应第 9、10 章，讲高级语言、编译链接、操作系统和系统资源管理。

## 核心结论

- 高级语言提升了表达效率，但最终仍要变成机器码。
- 操作系统提供资源管理与抽象，让应用不必直接面对底层硬件细节。

## 关键框架

### 1. 源代码到可执行文件
- 编译器把高级语言翻译成目标代码，链接器再组合成可执行文件。

### 2. 进程与资源
- 操作系统调度进程、分配内存、管理文件和设备。

### 3. 抽象层价值
- OS 的核心价值是隔离复杂性，而不是提供界面。

## 使用提醒

- 解释“为什么同一段代码能跨平台 / 为什么程序需要安装”时，先讲编译和 OS 抽象。
