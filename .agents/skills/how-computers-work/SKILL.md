## 核心框架

### 1. 计算机首先是可编程的电子设备
- 书里对计算机的定义很朴素：它是能被编程、并执行逻辑指令的电子设备。
- “电子设备”解释了它为什么要遵守电学规律；“可编程”解释了它为什么能和固定功能机器区分开。
- 使用时先分清：你问的是电子层问题，还是可编程层问题。

### 2. 一切数字系统都建立在有限状态表示上
- 计算机不是直接存“照片”“音乐”“网页”，而是把它们编码成有限状态序列。
- 在现代计算机里，这组有限状态几乎总是 `0` 和 `1`。
- 使用时遇到抽象问题，先回到“这类数据被编码成什么位模式”。

### 3. 二进制是连接现实世界和机器世界的第一座桥
- 二进制不是“神秘数学”，而是让有限状态系统可靠表示数字、文本和逻辑的方式。
- 理解位、字节、编码、ASCII/Unicode、符号位和进位，是后面所有内容的前提。
- 使用时只要看不懂更高层问题，就先退回二进制表示。

### 4. 数字电路把逻辑变成物理行为
- 逻辑门、开关、晶体管、加法器、锁存器和触发器，把抽象逻辑翻译成电压与电流的受控变化。
- 这一步是“软件为什么最终能落到硬件上”的关键。
- 使用时可以把程序行为映射到“计算、存储、控制”三类基础电路能力。

### 5. 计算机硬件的核心是 CPU、内存和 I/O
- CPU 负责执行指令，主存负责保存程序与数据，I/O 负责连接外部世界。
- 书里强调：通用计算机的关键不在单个器件，而在这些组件如何围绕“程序执行”协同工作。
- 使用时优先问：这个问题属于处理、存储还是输入输出。

### 6. 软件只是被 CPU 解释的一组编码指令
- 机器码、汇编、高级语言不是彼此割裂的三样东西，而是同一件事的不同抽象层。
- 程序员写的是源代码，编译器/汇编器把它转换为 CPU 能执行的机器码。
- 使用时可以顺着 `源代码 -> 编译/链接 -> 可执行文件 -> 指令 -> 寄存器/内存` 这条链路排查。

### 7. 操作系统是资源管理层，不只是“桌面界面”
- 操作系统负责进程、内存、文件、设备和权限管理，为上层程序提供抽象与服务。
- 它不是硬件，也不是普通应用，而是夹在二者之间的系统软件。
- 使用时不要把操作系统等同于 UI，要优先从资源与抽象的角度理解它。

### 8. 互联网和 Web 是两层不同系统
- 互联网是网络的网络，重点在 TCP/IP、分层协议、寻址和传输。
- Web 是跑在互联网之上的资源系统，重点在 URL、HTTP、浏览器和服务器。
- 使用时先分清：你遇到的是网络层问题，还是 Web 应用层问题。

### 9. 现代计算机是多层抽象叠加的结果
- app、虚拟化、云计算、网络 app 等，都是在前面几层基础上叠出来的现代形态。
- 理解现代系统时，最有效的方法不是记术语，而是回到底层抽象链条。
- 使用时把新概念映射到旧框架：它属于数据表示、执行、系统、网络还是分发层。

### 10. 最好的学习方式是“按层上楼，按层下钻”
- 这本书的强项是概念梯子：从模拟/数字 -> 二进制 -> 电路 -> CPU -> 编程 -> OS -> 网络 -> Web。
- 真正掌握不是背定义，而是能在解释时上下切换层次。
- 使用时默认先判断对方卡在哪一层，再补上一层和下一层的连接。

## 场景工作流

1. 先判断当前问题属于 `数据表示`、`数字电路`、`硬件执行`、`编程语言`、`操作系统`、`网络` 还是 `Web`。
2. 把问题放回对应抽象层，避免一上来混讲全部层次。
3. 必要时向下追一层：软件问题追到指令，指令问题追到硬件，网络问题追到协议。
4. 必要时向上提一层：位与电路如何影响 CPU，CPU 如何影响程序，程序如何依赖 OS 和网络。
5. 最后把结论翻译成可理解的比喻、实践项目或排查步骤。
