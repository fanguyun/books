# Books 项目指南

本文件用于约束 `/Users/fan/Downloads/电子书/books` 目录下的书籍整理、README 维护和 skills 维护行为。

## 1. 适用范围

- 本指南适用于 `books` 根目录、各书籍分类目录、`README.md`、`.agents/skills/` 下的本地 skills。
- 在本目录中新增、移动、删除书籍文件，或新增、修改 skills 时，均应同时检查 `README.md` 是否需要同步更新。
- 原始电子书文件默认视为“本地素材”，不应作为远程仓库常规提交内容。

## 2. 核心维护要求

### 2.1 新增书籍时

- 必须同步更新根目录 `README.md` 的“书籍目录”部分。
- 更新内容至少包括：
  - 书籍所属分类
  - 书籍文件名
- 如果新增书籍不适合现有分类，先评估是否需要新增分类，再写入 `README.md`。
- 新增的 `.epub`、`.mobi`、`.pdf` 默认只保留在本地工作区，不应纳入远程仓库版本控制，除非用户明确要求。

### 2.2 书籍移动、重命名、删除时

- 必须同步更新 `README.md` 中对应的书籍条目。
- 禁止出现目录结构与 `README.md` 不一致的状态。
- 如果此前误将原始电子书加入 Git，后续应从索引中移除，但保留本地文件。

### 2.3 skills 新增或变更时

- 只要 `.agents/skills/` 下有以下变更之一，就必须同步更新 `README.md` 的“已有 Skills”部分：
  - 新增 skill
  - 删除 skill
  - skill 名称变更
  - skill 路径变更
  - skill 用途发生明显变化
- 更新内容至少包括：
  - skill 名称
  - 相对路径
  - 简短作用说明
  - 典型用途说明

## 3. README 维护规则

- `README.md` 是本项目的统一索引，不是可选文档。
- 每次书籍目录或 skills 目录发生变更后，都要检查 `README.md` 是否仍然准确。
- 如果无法在当前变更中同步更新 `README.md`，应视为任务未完成。
- `README.md` 中的分类名称应与真实目录结构保持一致。
- `README.md` 应明确区分“本地保留的原始书籍”和“可推送到远程的索引/skills”。

## 4. 书籍目录划分建议

当前分类可继续沿用：

- `技术类`
- `心理学`
- `教育类`
- `育儿`
- `经济学`
- `成功学`
- `社会学`
- `国学`

后续新增书籍时，优先按“内容主题”而非作者、语言或文件格式划分。

推荐的分类判断顺序：

1. 先看书的主要主题，而不是副主题。
2. 如果一本书同时跨多个领域，归到最主要、最稳定的那个主题目录。
3. 不要为了单本书建立过细的小分类。
4. 当某一主题书籍明显增多，再新增目录。

可考虑的后续候选分类：

- `历史`
- `哲学`
- `文学`
- `传记`
- `商业`
- `管理`
- `投资`
- `产品`
- `设计`
- `AI`

新增分类时，必须同时满足：

- 现有分类明显不适合容纳该书
- 该分类后续大概率不是一次性使用
- `README.md` 已同步增加该分类小节

## 5. Skills 维护建议

- 优先保留一个“完整知识库版” skill，再按需要补充“中文别名入口”或“实战版”入口。
- 别名型 skill 应明确指向主 skill，避免内容重复维护。
- 场景型 skill 应尽量复用主 skill 的框架和 supporting files。
- 如果主 skill 结构变化导致别名或教练版引用失效，必须一并修复，并更新 `README.md`。

## 6. 变更原则

- 最小变更：只改当前任务直接相关的目录、README 和 skill 文件。
- 先改结构，再改索引：如果先移动文件，随后必须立即更新 `README.md`。
- 任何会影响查找入口的变更，都应以“人能否只看 README 找到书和 skill”为验收标准。

## 7. 验收标准

完成一次书籍或 skill 相关变更后，至少满足以下条件：

- 实际目录结构正确
- `README.md` 目录索引与实际一致
- `README.md` skills 索引与 `.agents/skills/` 实际一致
- 新增或变更后的入口路径可被直接定位
